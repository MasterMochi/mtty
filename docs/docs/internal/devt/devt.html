概要

デバイスイベントモジュールは、デバイスファイルを開いてデバイスファイルの読書きを監視し、読書きが可能になった時にイベントを発行する。

機能 - 初期化

初期化時にデバイスファイルオープンを行う。

シーケンスを以下に示す。

機能 - 初期化 - デバイスファイルオープン

デバイスファイルのオープンは、mvfsライブラリのLibMvfsOpen()を用いる。

引数 # 設定値 意味

2 "/serial1" シリアルポート1
3 (変数ポインタ) エラー要因

引数 # 設定値 説明
1 (変数ポインタ) ファイルディスクリプタ
1 "/serial2" シリアルポート2

機能 - 制御開始

デバイスファイルを監視し、デバイスが読書き可能になった時にイベント発行を行う。

シーケンスを以下に示す。

機能 - 制御開始 - デバイスファイル監視

デバイスファイルの監視は、監視するデバイスファイルのFD（ファイルディスクリプタ）を読込FDリストと書込FDリストに設定し、作成したFDリストを用いて監視する。


まず、読込FDリストと書込みFDリストをC標準ライブラリのmemset()を用いて初期化する。


FDリストの初期化
関数 memset()
引数 # 設定値 説明
1 (変数ポインタ) 読込FDリスト
書込FDリスト
2 0 初期化値
3 sizeof ( LibMvfsFds_t ) FDリストサイズ

次に、mvfsライブラリのLIBMVFS_FDS_SET()マクロを使用してFDリストへFDを設定する。

FDはデバイスファイルオープンに用いたLibMvfsOpen()の戻り値を使用する。


FDリストへのファイルディスクリプタ追加
マクロ LIBMVFS_FDS_SET()
引数 # 設定値 説明
1 (変数ポインタ) 読込FDリスト
書込FDリスト
2 (変数) シリアルポート1のFD
シリアルポート2のFD

書込みFDリストへのFDの追加は、書込み用バッファが空でない場合に行う。

書込み用バッファにデータが有るか否かは、バッファ管理モジュールのBufferCheckWrite()を用いて行う。


書込み用バッファのデータ有無判定
関数 BufferCheckWrite()
戻り値 データの有無を返す。
falseの場合は書込みFDリストへのFD追加は行わない。
引数 # 設定値 説明
1 MTTY_DEVID_SERIAL1 シリアルポート1用バッファチェック時
MTTY_DEVID_SERIAL2 シリアルポート2用バッファチェック時

最後に、mvfsライブラリのLibMvfsSelect()を用いてデバイスファイルの監視を開始する。

各FDリストに設定した何れかのFDが読込可能若しくは書込み可能な状態である時、そのFDがFDリストに再設定されて関数が戻る。


デバイスファイルの監視
関数 LibMvfsSelect()
引数 # 設定値 説明
1 (変数ポインタ) 読込FDリスト
2 (変数ポインタ) 書込FDリスト
3 50 タイムアウト時間[us]
3 (変数ポインタ) エラー要因

機能 - 制御開始 - イベント発行

mvfsライブラリのLibMvfsSelect()から制御が戻った時、各FDリストにFDが設定されているかmvfsライブラリのLIBMVFS_FDS_CHECK()マクロを用いて判定し、設定されていたFDに対応するデバイス制御のイベント処理関数を呼び出す。


読み書き可能ファイルディスクリプタの判定
マクロ LIBMVFS_FDS_CHECK
引数 # 設定値 説明
1 (変数ポインタ) 読込FDリスト
書込FDリスト
2 (変数) シリアルポート1のFD
シリアルポート2のFD

判定優先度と対応関数
#監視FDリストファイルディスクリプタ対応関数
1読込FDリストシリアルポート1DctrlDoRead()
2シリアルポート2
3書込FDリストシリアルポート1DctrlDoWrite()
4シリアルポート2
