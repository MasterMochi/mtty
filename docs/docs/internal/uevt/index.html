ユーザイベントモジュール

ユーザイベントモジュールは下記機能を持つ。

  1. 初期化
    1. スタック領域作成
    2. スレッド作成
    3. ターミナルファイル作成
  2. ターミナルファイル操作要求待ち合わせ
    1. ターミナルファイル操作要求待ち合わせ

初期化

mtty起動時に、スレッドを作成し作成したスレッド上でターミナルファイルの作成を行う。シーケンスを以下に示す。


シーケンス


スタック領域作成

ターミナルファイル制御用スレッドのスタック領域をC標準ライブラリのmalloc()を用いて割当て、memset()を用いて初期化する。設定パラメータを以下に示す。


malloc()設定パラメータ
# 引数名 設定値
1 size 8,192

memset()設定パラメータ
# 引数名 設定値
1 *s (スタック領域ポインタ)
2 c 0
3 n 8,192


スレッド作成

MochiKernelライブラリのLibMkThreadCreate()を用いて、ターミナルファイル制御用のスレッドを作成する。設定パラメータを以下に示す。


LibMkThreadCreate()設定パラメータ
# 引数名 設定値
1 pFunc (スレッドエントリ関数ポインタ)
2 *pArg NULL
3 *pStackAddr (スタック領域ポインタ)
4 stackSize 8,192
5 *pTaskId (タスクID格納先ポインタ)
6 *pErr (エラー要因格納先ポインタ)


ターミナルファイル作成

mttyライブラリのLibMvfsMount()を用いてターミナルファイルを作成する。設定パラメータを以下に示す。


LibMvfsMount()設定パラメータ
# 引数名 設定値
シリアルポート1 シリアルポート2
1 *pPath "/ttyS1" "/ttyS2"
2 *pErrNo (エラー要因格納先ポインタ)

ターミナルファイル操作要求待ち合わせ

初期化を行った後にターミナルファイル操作要求の待ち合わせを行う。シーケンスを下記に示す。


シーケンス


ターミナルファイル操作要求待ち合わせ

待ち合わせは、mvfsライブラリのLibMvfsSchedStart()を用いる。設定パラメータを以下に示す。


LibMvfsSchedStart()設定パラメータ
# 引数名 設定値
1 *pInfo (スケジュール情報ポインタ)
2 *pErrNo (エラー要因格納先ポインタ)

スケジュール情報
# 設定 設定値
1 callBack.pVfsOpen (open要求処理関数ポインタ)
2 callBack.pVfsWrite (write要求処理関数ポインタ)
3 callBack.pVfsRead (read要求処理関数ポインタ)
4 callBack.pVfsClose (close要求処理関数ポインタ)
5 callBack.pOther NULL
